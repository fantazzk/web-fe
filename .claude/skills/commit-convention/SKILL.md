# 커밋 메시지 규칙

이 프로젝트는 한글 커밋 메시지를 사용한다. 이모지는 사용하지 않는다.

## 형식

```
<타입>(<범위>): <제목>

<본문 (최대 3줄)>
```

## 타입

| 타입   | 용도                          |
| ------ | ----------------------------- |
| 기능   | 새로운 기능 추가              |
| 수정   | 버그 수정                     |
| 개선   | 기존 기능 개선, 리팩토링      |
| 설정   | 빌드, 설정, 의존성 변경       |
| 문서   | 문서 추가/수정                |
| 테스트 | 테스트 추가/수정              |
| 정리   | 코드 정리, 불필요한 코드 삭제 |

## 규칙

- 제목은 50자 이내로 간결하게 작성한다
- 제목은 명사형 종결로 끝낸다 (예: "추가", "수정", "변경")
- 본문은 최대 3줄까지만 작성한다
- 본문은 제목만으로 의도가 충분히 전달되지 않을 때만 작성한다
- 범위(scope)는 변경이 집중된 모듈이나 영역을 적는다 (생략 가능)
- 이모지를 사용하지 않는다

## 예시

**제목만:**

```
설정: TypeScript 엄격 모드 옵션 추가
```

**본문 포함:**

```
기능(인증): 소셜 로그인 추가

Google, GitHub OAuth 연동
기존 이메일 로그인과 병합 처리 포함
```

**범위 생략:**

```
수정: 목록 페이지 무한스크롤 중복 요청 방지
```
